Modèles de Classification
Régression Logistique
  • Classification binaire
  • Sortie : probabilité [0,1]
  • Modèle linéaire
Code
from sklearn.linear_model import LogisticRegression
model = LogisticRegression(max_iter=1000)
model.fit(X_train, y_train)

Quand la relation variables/probabilité est plutôt linéaire.

Arbre de Décision
  • Décisions selon les features
  • Facile à interpréter
  • Peut sur-apprendre
Code
from sklearn.tree import DecisionTreeClassifier
model = DecisionTreeClassifier(random_state=42)
model.fit(X_train, y_train)

Modèle interprétable avec relations non linéaires.

Random Forest
  • Ensemble de plusieurs arbres
  • Robuste, réduit l'overfitting
  • Bonne performance par défaut
Code
from sklearn.ensemble import RandomForestClassifier
model = RandomForestClassifier(
  n_estimators=100, random_state=42)
model.fit(X_train, y_train)

Par défaut dans la plupart des projets. Bon équilibre.

KNN
  • k voisins les plus proches
  • Simple à comprendre
  • Sensible au scaling
Code
from sklearn.neighbors import KNeighborsClassifier
model = KNeighborsClassifier(n_neighbors=5)
model.fit(X_train, y_train)

Dataset pas trop grand, données bien distribuées.

SVM
  • Trouve l'hyperplan optimal
  • Performant sur petits datasets
  • Sensible au choix du kernel
Code
from sklearn.svm import SVC
model = SVC(kernel='rbf', probability=True)
model.fit(X_train, y_train)

Classes bien séparables, linéairement ou non.

Naive Bayes
  • Basé sur le théorème de Bayes
  • Très rapide à entraîner
  • Idéal pour le texte (spam, NLP)
Code
from sklearn.naive_bayes import GaussianNB
model = GaussianNB()
model.fit(X_train, y_train)

Classification de texte, données indépendantes.

Matrice de Confusion
PRÉDIT
RÉEL 0 Négatif 1 Positif
0 Nég. TNTrue Negative FPFalse Positive
1 Pos. FNFalse Negative TPTrue Positive
Vrais (TN / TP) TN : Réel=0, Prédit=0 · TP : Réel=1, Prédit=1 · Bonne prédiction
Erreurs (FP / FN) FP : Réel=0, Prédit=1 · FN : Réel=1, Prédit=0 · Mauvaise prédiction
Accuracy sur données déséquilibrées 99 Non Fraude + 1 Fraude → Accuracy 99% mais aucune fraude détectée. Toujours regarder la matrice.
Métriques & Fonctions
Precision
TP / (TP + FP)
Faux positifs coûtent cher (spam, blocage carte)
Recall
TP / (TP + FN)
Faux négatifs graves (diagnostic, fraude)
F1 Score
2 × (P × R) / (P + R)
Compromis P/R · classes déséquilibrées
Accuracy
(TP + TN) / Total
A éviter sur données déséquilibrées

VALIDATION CROISÉE
cross_val_score
from sklearn.model_selection import cross_val_score
scores = cross_val_score(model, X, y, cv=5, scoring='f1')
scores.mean() # score moyen F1
Pourquoi ? Évalue le modèle sur plusieurs partitions pour vérifier sa stabilité et sa capacité de généralisation.
