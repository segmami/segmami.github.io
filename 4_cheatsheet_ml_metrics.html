Echelle de Valeur

Toutes les metriques retournent une valeur entre 0.0 et 1.0.

0 — mauvais 0.5 1 — parfait
ValeurInterpretation generale
Proche de 1Meilleur resultat — sauf si classes desequilibrees
Proche de 0Moins bon resultat
Exception : Accuracy trompeuse
Classes desequilibrees : 95% negatif / 5% positif
Predire toujours "non malade" donne Accuracy = 0.95 mais Recall = 0.00
Le modele ne detecte rien, pourtant l'accuracy est excellente.
Formules — Calcul avec l'exemple maladie (TP=5, FP=3, FN=5, TN=87, total=100)
Accuracy
(TP + TN) / total
(5 + 87) / 100 = 92/100
= 0.92 (92%)
Precision
TP / (TP + FP)
5 / (5 + 3) = 5/8
= 0.625 (62,5%)
Recall (Rappel)
TP / (TP + FN)
5 / (5 + 5) = 5/10
= 0.50 (50%)
F1-Score
2 * (P * R) / (P + R)
2*(0.625*0.5)/(0.625+0.5)
= 0.556 (55,6%)
Calcul des Metriques
python
# Accuracy : % bonnes predictions globales
print("Accuracy  :", accuracy_score(y_test, y_pred))   # 0.92

# Precision : parmi pred positifs, combien sont vrais
print("Precision :", precision_score(y_test, y_pred)) # 0.625

# Recall : parmi vrais positifs, combien detectes
print("Recall    :", recall_score(y_test, y_pred))    # 0.50

# F1-score : equilibre entre precision et recall
print("F1-Score  :", f1_score(y_test, y_pred))        # 0.556

Exemple complet — Interpretation des resultats (meme exemple : TP=5, FP=3, FN=5, TN=87)
MetriqueValeur calculeeResultatInterpretation
Accuracy 0.92 (92%) Bon en apparence 92% des predictions sont correctes — mais trompeuse car seulement 10 malades sur 100
Precision 0.625 (62,5%) Moyen Parmi les personnes predites malades, 62,5% sont reellement malades — 37,5% sont de faux positifs
Recall 0.50 (50%) Problematique Le modele rate 50% des vrais malades — 5 malades sur 10 passent inapercus
F1-Score 0.556 (55,6%) Faible Equilibre precision/recall revele la vraie faiblesse du modele malgre l'accuracy de 92%

Accuracy = 92% mais F1 = 55,6% et Recall = 50% — preuve qu'une bonne accuracy masque de mauvaises performances. Toujours verifier Recall et F1 en plus de l'Accuracy.

Precision — Quand utiliser

Utiliser quand les FP sont graves.

Parmi les personnes predites malades, combien sont vraiment malades ?

Spam — email normal marque spam = perte d'un message important. FP couteux : mieux vaut laisser passer un spam que bloquer un vrai email.
Test medical — dire "malade" a une personne saine = stress, examens inutiles. FP couteux : l'erreur a un impact meme si la personne est saine.
Recall — Quand utiliser

Utiliser quand les FN sont graves.

Parmi les vrais malades, combien le modele a reussi a detecter ?

Cancer — manquer un malade retarde le traitement. FN couteux : ne pas detecter un cas positif est grave.
Anti-fraude — laisser passer une fraude entraine des pertes d'argent. FN couteux : chaque faux negatif = une fraude non detectee.
Accuracy — Quand utiliser

Uniquement si les classes sont equilibrees.

Classes equilibrees — meme nombre de positifs et negatifs. L'accuracy reflecte bien la performance globale du modele.
Classes desequilibrees — 95 non malades / 5 malades :
Predire toujours "non malade" → Accuracy = 0.95 mais Recall = 0.00 (TP=0, FN=5, TN=95, FP=0)
Le modele ne detecte aucun malade, pourtant son accuracy semble excellente. Preferer Recall ou F1 dans ce cas.
