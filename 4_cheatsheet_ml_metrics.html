Calcul des Métriques
python
# % bonnes prédictions globales
accuracy_score(y_test, y_pred)   # 0.92
# parmi pred positifs, combien vrais
precision_score(y_test, y_pred)  # 0.625
# parmi vrais positifs, combien détectés
recall_score(y_test, y_pred)     # 0.50
# équilibre précision / recall
f1_score(y_test, y_pred)         # 0.556
Formules · exemple maladie (TP=5, FP=3, FN=5, TN=87, total=100)
Accuracy
(TP + TN) / total
(5 + 87) / 100
= 0.92 (92%)
Precision
TP / (TP + FP)
5 / (5 + 3) = 5/8
= 0.625 (62,5%)
Recall (Rappel)
TP / (TP + FN)
5 / (5 + 5) = 5/10
= 0.50 (50%)
F1-Score
2·(P·R)/(P+R)
2·(0.625·0.5)/(1.125)
= 0.556 (55,6%)

Échelle de Valeur

Toutes les métriques retournent une valeur entre 0.0 et 1.0.

0 · mauvais0.51 · parfait
Exception : Accuracy trompeuse
Même avec une Accuracy élevée (92%), le modèle peut manquer la moitié des malades (Recall = 50%). Exemple : 100 patients → TP=5, FP=3, FN=5, TN=87. Il semble performant grâce au grand nombre de non-malades bien classés (TN), mais ne détecte que 5 malades sur 10. Si les classes sont déséquilibrées, privilégier le Recall ou le F1-score plutôt que l'Accuracy seule.

Precision · Quand l'utiliser

Quand les FP sont graves.

Parmi les prédits malades, combien le sont vraiment ?

Spam : email normal marqué spam = perte d'un message important.FP coûteux : mieux vaut laisser passer un spam que bloquer un vrai email.
Test médical : dire « malade » à une personne saine = stress, examens inutiles.FP coûteux : l'erreur a un impact même si la personne est saine.
Recall · Quand l'utiliser

Quand les FN sont graves.

Parmi les vrais malades, combien sont détectés ?

Cancer : manquer un malade retarde le traitement.FN coûteux : ne pas détecter un cas positif est grave.
Anti-fraude : laisser passer une fraude entraîne des pertes.FN coûteux : chaque faux négatif = une fraude non détectée.
Accuracy · Quand l'utiliser
Classes équilibrées : même nombre de positifs et négatifs.L'accuracy reflète bien la performance globale du modèle.
Classes déséquilibrées : la classe majoritaire domine le score.Une accuracy élevée peut masquer un mauvais Recall. Préférer Recall ou F1.
