Workflow d'équipe, le flux complet

Règle d'or : jamais de commit direct sur dev ni main. Chacun sur sa branche, puis Pull Request. 1re fois : git clone <url> puis cd projet.
1
Synchroniser dev
dev
git switch dev
git pull origin dev
2
Créer sa branche
feature
git switch -c feature/tache
3
Travailler et commit
work
git add mes-fichiers
git commit -m "feat: tâche"
4
Récupérer le dev à jour (évite les conflits)
rebase
git fetch origin
git rebase origin/dev
# si conflit :
git add fichier-resolu
git rebase --continue
5
Pousser → Pull Request
push + PR
git push -u origin feature/tache
# GitHub : PR vers dev
6
PR fusionnée dans dev
merged
git switch dev
git pull origin dev
7
dev → main (production, lead)
release
git switch main
git merge dev
git push origin main
Option : fusion dev → main via PR pour tracer chaque mise en production.

Conflit de fusion

Deux branches modifient le même endroit. Git marque les zones ainsi :

marqueurs conflit
<<<<<<< HEAD
ta version
=======
version entrante
>>>>>>> entrante
résoudre fix
# retirer <<< === >>>
git add fichier-resolu
# merge :
git commit -m "Résolution"
# rebase :
git rebase --continue
git rebase --abort
